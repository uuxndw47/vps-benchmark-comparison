## 先说背景：两家都是德国公司，但路线不一样

Hetzner 成立于 1997 年，是欧洲老牌服务商，以稳定、透明、文档友好著称，深受开发者社区喜爱。

Contabo 成立于 2003 年，主打低价高配，核心卖点就是：同样的钱，我给你更多 CPU 核心、更多 RAM、更多带宽。

两家都是做 VPS 和独立服务器的，都在欧洲，但用户体验和性能表现差异相当明显。

---

## 性能对比：账面数据 vs 真实体验

这是整个 Hetzner vs Contabo 讨论里争议最大的地方。

**Contabo 的账面配置确实很诱人。** 你花同样的钱，在 Contabo 可能拿到 6 个虚拟核心、24GB RAM；而 Hetzner 同价位可能只有 2 个核心、8GB RAM。纸面上看，Contabo 完胜。

但是——

独立跑分数据说的是另一回事。多个第三方性能评测（包括 Gart Solutions、VPSBenchmarks 等）的测试结果显示，Hetzner 的单核性能大约是 Contabo 的 3 倍左右。这个差距在 Web 框架、WordPress、数据库这类高度依赖单线程的应用里，体感非常明显。

原因也不复杂：Contabo 的资源超售比例更高，你那 6 个"虚拟核心"要跟更多邻居共享同一台物理机。高峰期 CPU steal time（资源被其他租户偷走的比例）会拉高，性能就不稳了。

Hetzner 的 CCX（专用 CPU）系列解决了这个问题，但价格也相应上来了。

**简单总结：**

| 维度 | Hetzner | Contabo |
|------|---------|---------|
| 账面配置 | 保守，实际能用到的 | 激进，纸面好看 |
| 单核性能 | 约 3× 高于 Contabo | 偏低，稳定性差 |
| CPU 一致性 | 高，波动小 | 较高波动，峰值时段明显 |
| 磁盘 I/O | NVMe，表现稳定 | 有 NVMe 可选，但表现不一致 |
| 部署速度 | 平均 30 秒以内 | 超过 100 秒 |

---

## 价格对比：谁更便宜要看你怎么算

Contabo 胜在账面资源的价格比。如果你需要大量 RAM 和存储，比如跑 Java 应用、大型数据库、或者内存密集型任务，Contabo 的性价比确实高。

Hetzner 的定价逻辑不一样。它提供按小时计费（更灵活），流量超额按固定费率收取（1 欧/TB，透明无套路），总体用起来没有隐性费用。对于流量大的应用，这个模型往往比 Contabo 省钱。

举一个实际案例：一个每月传输 100TB 数据的应用，在一些按流量计费的平台上，超额费用会让你眼泪流下来。Hetzner 欧区的超额流量费相对实惠，这让它对流量密集型业务更友好。

**定价维度对比：**

| 维度 | Hetzner | Contabo |
|------|---------|---------|
| 计费方式 | 按小时 / 按月 | 主要按月 |
| 流量超额 | 约 €1/TB，透明 | 欧洲节点最多 32TB，超出后限速 |
| 入门价格 | ~$4-5/月（2 核 4GB） | 约 $5-6/月（4 核 8GB） |
| 独立 CPU 方案 | CCX 系列，价格更高 | VDS 系列，配置灵活 |

---

## 服务位置：Contabo 全球，Hetzner 主力欧洲

这是 Contabo 的一个真实优势。它在北美、欧洲、亚太都有节点（包括新加坡、日本等），对亚洲用户来说覆盖更广。

Hetzner 的主要数据中心在德国、芬兰，以及美国弗吉尼亚和俄勒冈，亚太区没有节点。如果你的目标用户在东南亚、东亚，Hetzner 的延迟会成问题。

这个问题也是很多亚太开发者最终没选 Hetzner 的核心原因之一。

---

## 支持与文档：Hetzner 明显更好

用过两家的人基本都这么说。Hetzner 的文档质量高，工单响应快（通常几小时内），有 Terraform 官方支持，API 文档规范，是 IaC（基础设施即代码）实践者的首选。

Contabo 的支持是个老问题：邮件为主，响应慢，文档相对欠缺。这不是用户的偏见，而是用户评测里反复出现的一致反馈。

---

## Hetzner vs Contabo 适合人群一句话版

- **选 Hetzner** 如果你：做生产环境、跑数据库、需要稳定单核性能、在意 API 和自动化、用户在欧美。
- **选 Contabo** 如果你：预算极度有限、需要大量 RAM 或存储、面向亚洲用户且需要全球节点、对性能波动有一定容忍度。

## 把三者放在一起看

说到底，Hetzner、Contabo、Evoxt 针对的是不同场景下的不同需求：

- **Hetzner**：生产环境、欧洲用户、注重单核性能和稳定性，愿意为一致性多付一点。
- **Contabo**：预算紧张、需要大配置跑内存密集型任务、对性能波动有容忍度，主要跑欧美或全球分布业务。
- **Evoxt**：亚太为主或全球需求、追求高主频单核性能、不想在低价 VPS 里牺牲实际跑分，尤其是需要香港 CN2、日本、东南亚节点的用户。

三家的起步价都在 $3-6/月 区间，最低门槛差别不大，关键是看你把流量、延迟、CPU 频率、哪个地区这几个变量放在什么优先级。
